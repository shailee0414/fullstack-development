⚙ Core JavaScript
01

Execution Context

Fundamental

An Execution Context (EC) is the environment in which JavaScript code is evaluated and executed. Every time the JS engine runs code, it creates an EC. There are three types: Global EC, Function EC, and Eval EC.

Each EC has two phases: Creation Phase (sets up memory — variables are hoisted, `this` is bound, outer environment is linked) and Execution Phase (code runs line by line).

Creation Phase

Variable Environment created, arguments object set, `this` binding set, hoisting happens (var → undefined, functions → full definition)

Execution Phase

Code executes line by line. Variables get assigned real values. Functions get called, spawning new ECs on the Call Stack.

execution-context.js
// Global EC is created first
var name = 'Alice';

function greet() {
  // New Function EC is pushed onto Call Stack
  var msg = 'Hello, ' + name;
  console.log(msg); // "Hello, Alice"
  // Function EC is popped when function returns
}

greet();
// Call Stack: [Global EC] → [Global EC, greet EC] → [Global EC]
The Call Stack is a stack data structure that manages execution contexts. Global EC is always at the bottom. When a function is called, its EC is pushed; when it returns, it's popped.

Hoisting

Core

Hoisting is JavaScript's behavior of moving declarations to the top of their scope during the Creation Phase. Only declarations are hoisted — not initializations.

KeywordHoisted?Initial ValueTDZ?
var✅ YesundefinedNo
let✅ Yes (declaration)uninitialized✅ Yes
const✅ Yes (declaration)uninitialized✅ Yes
Function Declaration✅ Fully hoistedFull functionNo
Function ExpressionPartially (var part)undefined—
hoisting.js
console.log(x); // undefined (var is hoisted)
var x = 5;

sayHi(); // "Hi!" — function declaration fully hoisted
function sayHi() { console.log('Hi!'); }

// console.log(y); → ReferenceError: TDZ (Temporal Dead Zone)
let y = 10;

// sayBye(); → TypeError: sayBye is not a function
var sayBye = function() { console.log('Bye!'); };
TDZ (Temporal Dead Zone): The period between the start of a block and the point where let/const is initialized. Accessing the variable in TDZ throws a ReferenceError.

03

Scope

Core

Scope determines where variables and functions are accessible. JavaScript uses Lexical Scoping — scope is determined at write time, not runtime.

Global Scope

Variables declared outside any function/block. Accessible everywhere. var becomes a property of window.

Function Scope

Variables declared with var inside a function. Accessible only within that function.

Block Scope

let and const are scoped to the nearest { } block (if, for, while).

Lexical / Static Scope

Functions remember the scope where they were defined, not where they're called. Basis of closures.

scope.js
var global = 'global';

function outer() {
  let outerVar = 'outer';

  function inner() {
    const innerVar = 'inner';
    console.log(global, outerVar, innerVar); // all accessible
  }

  inner();
  // console.log(innerVar); → ReferenceError
}

if (true) {
  var a = 1;  // function-scoped (leaks out of block)
  let b = 2;  // block-scoped
}
console.log(a); // 1
// console.log(b); → ReferenceError

Closures (Deep Dive)

Must Know

A closure is a function that remembers and accesses variables from its outer (lexical) scope even after that outer function has finished executing. Closures are created every time a function is created.

Think of a closure as a function bundled with its surrounding state (lexical environment). The inner function "closes over" the outer variables.
closure-basics.js
function makeCounter() {
  let count = 0; // private variable
  return {
    increment: () => ++count,
    decrement: () => --count,
    value:     () => count
  };
}

const counter = makeCounter();
counter.increment(); // 1
counter.increment(); // 2
counter.decrement(); // 1
console.log(counter.value()); // 1
// count is not directly accessible — it's private!
closure-loop-gotcha.js
// Classic interview problem — var vs let in loops
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100); // 3, 3, 3 (var is shared)
}

for (let j = 0; j < 3; j++) {
  setTimeout(() => console.log(j), 100); // 0, 1, 2 (let has block scope)
}

// Fix with var using IIFE:
for (var k = 0; k < 3; k++) {
  ((function(k) {
    setTimeout(() => console.log(k), 100);
  })(k));
}

Real-world uses: data privacy, memoization, partial application/currying, module pattern, event handlers.

this refers to the execution context — the object that is currently calling the function. Its value depends on how a function is called, not where it's defined (except for arrow functions).

Type Coercion

Gotcha

Type Coercion is JavaScript automatically converting one type to another. It happens with ==, arithmetic operators, template literals, and boolean contexts. Can be implicit (auto) or explicit (manual).

coercion.js
// Implicit coercion
'5' + 3    // "53"  (number → string)
'5' - 3    // 2     (string → number)
true + 1   // 2
false + 1  // 1
null + 1   // 1
'' == false // true

// Abstract Equality (==) with coercion
0 == false   // true
null == undefined // true (special case)
NaN == NaN   // false! (NaN is never equal to itself)

// Strict Equality (===) — no coercion
'5' === 5   // false

// Explicit conversion
Number('42')   // 42
String(42)    // "42"
Boolean(0)    // false
Boolean('')   // false
Boolean([])   // true! (empty array is truthy)

Core

All three methods allow you to explicitly set this for a function. The difference is in how arguments are passed and when the function executes.

MethodArgumentsExecutes Immediately?
call(ctx, a, b)Spread individually✅ Yes
apply(ctx, [a, b])As an array✅ Yes

  return `${greeting}, I'm ${this.name}${punct}`;
}

const alice = { name: 'Alice' };

introduce.call(alice, 'Hi', '!');        // "Hi, I'm Alice!"
introduce.apply(alice, ['Hello', '.']);  // "Hello, I'm Alice."

const boundFn = introduce.bind(alice, 'Hey');
boundFn('?'); // "Hey, I'm Alice?"

  const fn = this;
  return function(...newArgs) {
    return fn.apply(ctx, [...args, ...newArgs]);

Higher Order Functions

Important

A Higher Order Function (HOF) is a function that either takes a function as an argument or returns a function. They enable functional programming patterns in JS.

const nums = [1, 2, 3, 4, 5];

// Function returning function
const multiplier = n => x => x * n;
const double = multiplier(2);

Currying

Advanced

Currying transforms a function that takes multiple arguments into a sequence of functions each taking one argument. f(a, b, c) becomes f(a)(b)(c).

currying.js
// Manual curry
function add(a) {
  return function(b) {
    return function(c) {
      return a + b + c;
    };
  };
}
add(1)(2)(3); // 6

// Generic curry function (interview polyfill)
function curry(fn) {
  return function curried(...args) {
    if (args.length >= fn.length) {
      return fn.apply(this, args);
    }
    return function(...args2) {
      return curried.apply(this, args.concat(args2));
    };
  };
}

const curriedAdd = curry((a, b, c) => a + b + c);
curriedAdd(1)(2)(3); // 6
curriedAdd(1, 2)(3); // 6

Memoization

Performance

Memoization is a caching optimization technique — store the results of expensive function calls and return the cached result when the same inputs occur again. Uses closures under the hood.

memoization.js
function memoize(fn) {
  const cache = new Map();
  return function(...args) {
    const key = JSON.stringify(args);
    if (cache.has(key)) {
      console.log('Cache hit!');
      return cache.get(key);
    }
    const result = fn.apply(this, args);
    cache.set(key, result);
    return result;
  };
}

const expensiveAdd = (a, b) => { /* heavy computation */ return a + b; };
const memoAdd = memoize(expensiveAdd);
memoAdd(2, 3); // computed
memoAdd(2, 3); // Cache hit! (no recomputation)

Deep Equality

Interview

=== compares object references, not values. Deep equality checks if two objects have the same structure and values recursively.

deep-equal.js
function deepEqual(a, b) {
  if (a === b) return true;
  if (typeof a !== 'object' || typeof b !== 'object') return false;
  if (a === null || b === null) return false;

  const keysA = Object.keys(a), keysB = Object.keys(b);
  if (keysA.length !== keysB.length) return false;

  return keysA.every(key => deepEqual(a[key], b[key]));
}

deepEqual({ a: { b: 1 } }, { a: { b: 1 } }); // true
deepEqual([1, 2], [1, 2]);                  // true

Prototypes

Core

Every JavaScript object has an internal link to another object called its prototype. Objects inherit properties and methods from their prototype. This is prototype-based inheritance — different from class-based OOP.

prototypes.js
function Animal(name) {
  this.name = name;
}
Animal.prototype.speak = function() {
  return `${this.name} makes a sound.`;
};

const dog = new Animal('Dog');
dog.speak(); // "Dog makes a sound."

// dog.__proto__ === Animal.prototype → true
// Animal.prototype.__proto__ === Object.prototype → true
// Object.prototype.__proto__ === null (end of chain)

// Object.create() — create with specific prototype
const cat = Object.create(Animal.prototype);
cat.name = 'Cat';
cat.speak(); // "Cat makes a sound."

// ES6 Class Syntax (syntactic sugar over prototypes)

Event Loop (Deep Dive)

Must Know

JavaScript is single-threaded but handles concurrency via the Event Loop. The loop continuously checks if the Call Stack is empty, then moves tasks from queues to the stack.

Call Stack

Synchronous code runs here LIFO. One task at a time. If it blocks, everything freezes.

Web APIs

Browser-provided async handlers: setTimeout, fetch, DOM events. Offloaded here, not blocking the stack.

Microtask Queue

Higher priority. Promises (.then), queueMicrotask, MutationObserver. Drained completely before Macrotask.

Macrotask Queue

setTimeout, setInterval, I/O, UI rendering. One per event loop iteration after microtasks.

event-loop-order.js
console.log('1 - sync');

setTimeout(() => console.log('4 - macrotask'), 0);

Promise.resolve().then(() => console.log('3 - microtask'));

console.log('2 - sync');

// Output: 1, 2, 3, 4
// Sync → Microtasks → Macrotasks

async / await

Important

async/await is syntactic sugar over Promises, making async code look synchronous. An async function always returns a Promise. await pauses execution inside the function (not globally) until the Promise resolves.

async-await.js
async function getUser(id) {
  try {
    const res  = await fetch(`/api/user/${id}`);
    if (!res.ok) throw new Error(`HTTP ${res.status}`);
    const user = await res.json();
    return user;
  } catch (err) {
    console.error('Failed:', err);
    throw err;
  }
}

// Parallel with async/await
async function loadAll() {
  const [users, posts] = await Promise.all([
    fetch('/api/users').then(r => r.json()),
    fetch('/api/posts').then(r => r.json()),
  ]);
  return { users, posts };
}

Debouncing

Polyfill

Debounce delays executing a function until after a specified time has elapsed since the last call. Use case: search input — wait until user stops typing.

debounce.js
function debounce(fn, delay) {
  let timer;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  };
}

const onSearch = debounce((query) => {
  fetch(`/api/search?q=${query}`); // only fires 500ms after last keystroke
}, 500);

input.addEventListener('input', e => onSearch(e.target.value));

Throttling

Polyfill

Throttle ensures a function is called at most once per specified time period, regardless of how many times it's triggered. Use case: scroll handler, resize events.

throttle.js
function throttle(fn, limit) {
  let inThrottle = false;
  return function(...args) {
    if (!inThrottle) {
      fn.apply(this, args);
      inThrottle = true;
      setTimeout(() => inThrottle = false, limit);
    }
  };
}

const onScroll = throttle(() => {
  console.log('Scroll event (max once per 200ms)');
}, 200);

window.addEventListener('scroll', onScroll);
Debounce vs Throttle: Debounce → fires AFTER silence. Throttle → fires AT MOST every N ms. Both use closures.

  config.headers.Authorization = `Bearer ${getToken()}`;

  if (err.response?.status === 401) logout();

// Retry logic
async function fetchWithRetry(url, retries = 3) {
  for (let i = 0; i < retries; i++) {
    try {
      return await (await fetch(url)).json();
    } catch (e) {
      if (i === retries - 1) throw e;
      await new Promise(r => setTimeout(r, 1000 * (i + 1))); // exponential backoff
    }
  }
}

Virtual DOM & Reconciliation

React

The Virtual DOM is an in-memory representation of the real DOM. React creates a virtual tree, and when state changes, it creates a new virtual tree and diffs it against the old one — only updating what changed in the real DOM.

Diffing Algorithm

React compares old vs new VDOM trees. Same element type → update props. Different type → destroy and recreate. List items need unique key props.

React Fiber

React's reconciliation engine. Breaks rendering into units of work, allowing pausing, prioritization, and resuming — enabling features like Concurrent Mode.

Event Delegation & Bubbling

Core

Event Bubbling: events propagate from child → parent → document. Event Delegation: attach one listener on a parent instead of many listeners on children — handles dynamic elements efficiently.

event-delegation.js
// Without delegation — bad for dynamic lists
document.querySelectorAll('li').forEach(li =>
  li.addEventListener('click', handler)

// With delegation — efficient
document.getElementById('list').addEventListener('click', (e) => {
  if (e.target.matches('li')) {
    console.log('Clicked:', e.target.textContent);
  }
});

// Stop propagation
e.stopPropagation();   // stops bubbling
e.preventDefault();   // prevents default browser action

// Capture phase (parent first)
el.addEventListener('click', handler, { capture: true });

State Batching & Functional Updates

React
react-state.jsx
// State Batching (React 18+) — multiple setState calls in one render
function Counter() {
  const [count, setCount] = useState(0);
  const [flag, setFlag]   = useState(false);

  const handleClick = () => {
    setCount(c => c + 1); // functional update — based on previous
    setFlag(f => !f);
    // React 18 batches these → ONE re-render, not two
  };

  // ⚠ Wrong — reads stale state
  const bad = () => {
    setCount(count + 1); // count is stale closure
    setCount(count + 1); // still count+1, not count+2!
  };

  // ✅ Correct — functional form
  const good = () => {
    setCount(c => c + 1);
    setCount(c => c + 1); // correctly count+2
  };
}
In React 18, batching works everywhere — even inside setTimeout, promises, and native event handlers. Use flushSync() to opt out of batching when needed.

Event Propagation — Capturing vs Bubbling

Deep Dive

DOM events travel in 3 phases: Capture (window → target), Target (at element), Bubble (target → window). By default, handlers are registered on the bubble phase.

event-propagation.js
// Capture phase — parent first
parent.addEventListener('click', () => console.log('parent capture'), true);
child.addEventListener( 'click', () => console.log('child'));
parent.addEventListener('click', () => console.log('parent bubble'));
// Click child → "parent capture", "child", "parent bubble"

e.stopPropagation();      // stop at current element
e.stopImmediatePropagation(); // also stops other handlers on same element

XSS & CSRF

Security
XSS (Cross-Site Scripting)

Attacker injects malicious scripts into your page. Prevention: never use innerHTML with user input — use textContent or sanitize with DOMPurify. Set Content-Security-Policy header. Use HttpOnly cookies.

CSRF (Cross-Site Request Forgery)

Malicious site tricks authenticated user's browser into making requests to your API. Prevention: CSRF tokens, SameSite cookies (SameSite=Strict), check Origin/Referer headers.

// Array.prototype.map polyfill
Array.prototype.myMap = function(fn) {
  const result = [];
  for (let i = 0; i < this.length; i++) {
    result.push(fn(this[i], i, this));
  }
  return result;
};

// Array.prototype.filter polyfill
Array.prototype.myFilter = function(fn) {
  const result = [];
  for (let i = 0; i < this.length; i++) {
    if (fn(this[i], i, this)) result.push(this[i]);
  }
  return result;
};

// Array.prototype.reduce polyfill
Array.prototype.myReduce = function(fn, init) {
  let acc = init !== undefined ? init : this[0];
  let start = init !== undefined ? 0 : 1;
  for (let i = start; i < this.length; i++) {
    acc = fn(acc, this[i], i, this);
  }
  return acc;
};

// Promise.all polyfill
Promise.myAll = function(promises) {
  return new Promise((resolve, reject) => {
    const results = [];
    let count = 0;
    promises.forEach((p, i) => {
      Promise.resolve(p).then(val => {
        results[i] = val;
        if (++count === promises.length) resolve(results);
      }).catch(reject);
    });
  });
};
